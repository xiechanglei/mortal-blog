# xie-site

25年接近年尾一个阳光明媚的早晨，刚到公司，离上班还有一个小时，手里捧着一杯热乎乎的白开水，手边是刚买的感冒药，不知从前几年的什么时候开始，身体好像变的很脆弱，气温骤降下，稍不注意就感冒。

一直有做个博客的想法，偶尔记录一下生活，当然与工作无关，因为工作中的技术性文章，如果我要去写的话，一个是需要查阅大量的资料，因为技术是严谨的，另外一个是可能需要持续的跟进，我也没有那么多的时间和精力去维护一个技术博客，所以就想写一些生活中的琐事，记录一下自己的心情。

最近有看到使用vercel部署网站的一些文章，觉得挺有意思的，可以尝试做一下，整体的思路是使用markdown文件来编写文章，然后使用nodejs生成静态网站，最后部署到vercel上面去。

试试吧， 希望未来的某一天，翻阅这些文字的时候，能回忆起当时的心情和故事。

## 项目结构设计

项目遵循以下约定：

1. `articles` 目录下存放的是所有的markdown文章文件,我会在这里编写所有的文章
2. `build` 文件夹中有用于生成文章信息的脚本文件，生成的所有文章的信息会放在`data` 目录下
3. `data` 目录下存放的是生成的json文件，包括一个`index.json`文件，用以描述所有的文章信息，包括标题，文件目录，字数，修改时间，以及提取前面部分的文本信息和封面图等字段。封面图我会在markdown文件的头部进行定义
4. `index.html` 是网站的入口
5. `pages` 目录下存放的是网站的其他页面
6. `public` 目录下存放的是网站的静态资源文件

网页布局：

1. 主页主要分为两部分，上面是网页的标题栏，标题栏左侧是网站的名称`Mortal's Blog`，右侧是导航栏，导航栏的左侧是一个搜索框，用以搜索文章，导航栏的右侧是一些外部链接，比如github等
2. 主页的中间部分是文章列表，使用grid布局，每行显示最多4篇文章，按照文章的修改时间倒序排列，每篇文章显示标题，封面图，修改时间，字数，和文章的前几段内容
3. 点击文章标题或者封面图可以进入文章详情页面，文章详情页面显示文章的信息信息，使用fetch加载对应的markdown文件，然后使用markdown解析器进行渲染
